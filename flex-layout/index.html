弹性布局基础类

弹性布局详细教程： https://www.runoob.com/w3cnote/flex-grammar.html
一、基本介绍
1、优势简介：
（1）功能强大：能满足99%以上的布局需求（除了响应式布局、脱离文档流的定位）；轻松处理行内和行内块元素的间距问题。
（2）快捷高效：类名简短易记、代码量少。
（3）兼容性好（少踩坑）：PC端、移动端的兼容性都挺好。例：开发微信小程序时能少踩坑。
2、源文件：
Css文件、 Less文件、 Sass文件
3、基本属性：
justify-content：flex-start、flex-end、center、space-between、space-around、 space-evenly（兼容性不太好）
align-items：flex-start、flex-end、center、baseline、stretch
flex-direction：row、 row-reverse、 column、 column-reverse
flex-wrap：nowrap、 wrap、 wrap-reverse
align-content：flex-start、flex-end、center、space-between、space-around、 stretch
注：带删除线的是几乎用不到的属性，故类名中的对应位置不应该包含该属性的缩写。
4、使用示例及含义剖析：
注：
（1）类名最少包含3段字符，最多4段字符，用“-”进行连接，每段字符均取自对应属性的缩写。
（2）对于项目的伸缩特性，文本省略处理，记得带上f-0、f-1、line-1（不能跟 f-sb-c 等类名放在一起）等类名。
（3）第五段字符用时情况极少，会生成过多无用css代码，故不计划纳入，只是需要知道这个属性。如有特殊情况需要大量使用，将对应注释放开即可，使用示例：f-sb-fs-w-sb。

四、固定头部或底部，其他区域可滚动的场景
注：在黄色容器高度固定（即有确定的值，如：100vh，能够继承到的100%）的情况下：
1、黄色容器类：f-fs-s-c
描述：头部、底部固定高度（f-0），中间不设置高度，底部会跟随高度自适应变化。
场景举例：答题页面。问题栏高度固定，选项栏个数不定，超过最大高度出现滚动条，未超过，则上一下或下一题按钮跟随高度自适应变化。

overflow:auto

2、黄色容器类：f-fs-s-c
描述：头部、底部固定高度（f-0），中间伸缩高度（f-1），不管内容是否超过最大高度，底部始终位于页面最下方。
场景举例：顶部标题、底部按钮、中间内容高度均固定，且中间内容高度会自适应撑满屏幕高度，当内容超过最大高度时会出现滚动条。
